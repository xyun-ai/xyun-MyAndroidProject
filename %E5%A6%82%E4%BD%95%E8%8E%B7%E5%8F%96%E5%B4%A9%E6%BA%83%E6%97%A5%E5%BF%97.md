# 如何获取应用崩溃日志

如果应用仍然闪退，请按照以下步骤获取崩溃日志：

## 方法1：使用Android Studio的Logcat

1. 在Android Studio中打开项目
2. 连接设备或启动模拟器
3. 点击底部工具栏的 "Logcat" 标签
4. 在搜索框中输入 "AndroidRuntime" 或 "FATAL"
5. 运行应用，查看崩溃日志

## 方法2：使用ADB命令

1. 打开命令行终端
2. 导航到Android SDK的platform-tools目录
3. 运行以下命令清除旧日志：
   ```
   adb logcat -c
   ```
4. 运行以下命令开始记录日志：
   ```
   adb logcat -v time *:E > crash.log
   ```
5. 在另一个终端中运行应用
6. 应用崩溃后，按Ctrl+C停止日志记录
7. 查看crash.log文件中的错误信息

## 方法3：在代码中添加全局异常处理

在MainActivity.java的onCreate方法开始处添加：

```java
Thread.setDefaultUncaughtExceptionHandler(new Thread.UncaughtExceptionHandler() {
    @Override
    public void uncaughtException(Thread thread, Throwable throwable) {
        Log.e("CRASH", "应用崩溃:", throwable);
        throwable.printStackTrace();
    }
});
```

## 常见崩溃原因

1. **空指针异常** - 检查findViewById返回值是否为空
2. **资源找不到** - 检查drawable和layout资源
3. **主题不兼容** - 确保使用AppCompat主题
4. **权限问题** - 检查AndroidManifest.xml
5. **依赖冲突** - 检查build.gradle依赖
